## 📖 Быстрый старт (10 минут)

### PostgreSQL

```sql
-- 1. Создать VIEW
CREATE VIEW public.report_sales AS
SELECT customer, SUM(amount) as total
FROM orders
GROUP BY customer;

-- 2. Использовать в коде (C#)
var reportInfo = new ReportInfo { Name = "report_sales", Schema = "public", Type = "View" };
var data = await reportsRepository.ExecuteReportAsync(reportInfo);
```

### SQLite

```sql
-- 1. Создать таблицу отчёта
CREATE TABLE report_sales AS
SELECT customer, SUM(amount) as total
FROM orders
GROUP BY customer;

-- 2. Использовать в коде (C#)
var reportInfo = new ReportInfo { Name = "report_sales", Schema = "main", Type = "MaterializedView" };
var data = await reportsRepository.ExecuteReportAsync(reportInfo);
```

## 🔑 Ключевые концепции

### Типы отчётов

| Тип | PostgreSQL | SQLite | Когда использовать |
|-----|-----------|--------|-------------------|
| **View** | ✅ CREATE VIEW | ✅ CREATE VIEW | Часто меняются данные |
| **MaterializedView** | ✅ CREATE MATERIALIZED VIEW | ⚠️ CREATE TABLE AS SELECT | Большой объём, редко меняются |
| **Function** | ✅ Функция с параметрами | ❌ Не поддерживается | Параметризованные запросы |

### ATTACH DATABASE (SQLite)

```csharp
// Подключить внешние БД
reportInfo.AttachDatabases = "db1=D:\\data\\sales.db;db2=D:\\data\\products.db";

// SQL использует alias
// SELECT s.id, p.name FROM db1.sales s JOIN db2.products p ON ...
```

### Материализованные представления

```csharp
// Обновить кэшированные данные
await reportsRepository.RefreshMaterializedViewAsync(reportInfo);
```

## 📊 Производительность

### Выбор типа отчёта

```
Частые обращения (> 100/день)?
├─ Да → MaterializedView (с индексами)
└─ Нет → View

Большой объём данных (> 100K)?
├─ Да → MaterializedView
└─ Нет → View

Данные редко меняются (< 1 раз/день)?
├─ Да → MaterializedView
└─ Нет → View
```

### Оптимизация

1. **Выбирать только нужные колонки**
   ```sql
   SELECT id, name, amount -- не SELECT *
   ```

2. **Добавить индексы на часто фильтруемые колонки**
   ```sql
   CREATE INDEX idx_report_date ON report_sales(date DESC);
   ```

3. **Ограничить данные в запросе**
   ```sql
   WHERE date >= date('now', '-30 days') -- фильтруем на уровне БД
   ```
