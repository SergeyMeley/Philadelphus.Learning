# Система отчётов Philadelphus

## Обзор

Система отчётов позволяет создавать и выполнять запросы к данным через единый интерфейс, независимо от используемой СУБД (PostgreSQL, SQLite и т.д.).

## Ключевые возможности

✅ Единый интерфейс для всех СУБД
✅ Поддержка разных типов отчётов (представления, функции)
✅ Параметризованные отчёты
✅ Материализованные представления
✅ Кэширование результатов
✅ Документирование отчётов

## Архитектура

### Интерфейс IReportsInfrastructureRepository

Основной интерфейс для работы с отчётами:

```csharp
public interface IReportsInfrastructureRepository : IInfrastructureRepository
{
    // Получить список доступных отчётов
    Task<List<ReportInfo>> GetAvailableReportsAsync(string schemaName);
    
    // Выполнить отчёт и получить результаты
    Task<DataTable> ExecuteReportAsync(ReportInfo report);
    
    // Обновить материализованное представление
    Task RefreshMaterializedViewAsync(ReportInfo report);
}
```

### Модель ReportInfo

```csharp
public class ReportInfo
{
    // Название отчёта/представления/функции
    public string Name { get; set; }
    
    // Схема БД (для PostgreSQL) или 'main' для SQLite
    public string Schema { get; set; }
    
    // Тип: "View", "MaterializedView" или "Function"
    public string Type { get; set; }
    
    // Описание отчёта из комментариев БД
    public string Description { get; set; }
    
    // Параметры (для функций)
    public List<ReportParameter> Parameters { get; set; }
    
    // Информация об ATTACH БД (для SQLite)
    public string AttachDatabases { get; set; }
}
```

## Типы отчётов

### View (Представление)

- **Что:** Виртуальная таблица, создаваемая на основе SQL запроса
- **Где:** Создаётся в БД как обычное представление
- **Особенности:**
  - Не занимает место на диске
  - Выполняется при каждом обращении
  - Всегда возвращает актуальные данные
  - Может использовать индексы базовых таблиц

**Когда использовать:**
- Данные часто изменяются
- Объём данных небольшой
- Нужны самые свежие данные

### MaterializedView (Материализованное представление)

- **Что:** Физическая таблица, содержащая снимок данных
- **Где:** Создаётся в БД как таблица или специальное представление
- **Особенности:**
  - Занимает место на диске
  - Выполняется один раз (при создании/обновлении)
  - Возвращает кэшированные данные
  - Очень быстрое выполнение

**Когда использовать:**
- Данные изменяются редко
- Объём данных большой
- Нужна максимальная производительность

### Function (Функция)

- **Что:** Функция БД, возвращающая таблицу результатов
- **Где:** Создаётся в БД как функция (только PostgreSQL)
- **Особенности:**
  - Может принимать параметры
  - Выполняется при каждом обращении
  - Поддерживает сложную логику
  - Может использовать вычисления

**Когда использовать:**
- Нужны параметризованные отчёты
- Требуется сложная логика
- Нужны динамические фильтры

## Метаданные отчётов

### Таблица system_report_metadata (SQLite)

Система хранит метаданные в таблице `system_report_metadata`:

```sql
CREATE TABLE IF NOT EXISTS system_report_metadata (
    report_name TEXT PRIMARY KEY,
    attach_databases TEXT,
    creation_sql TEXT
);
```

**Колонки:**

| Колонка | Тип | Назначение |
|---------|-----|-----------|
| report_name | TEXT | Имя таблицы отчёта (PRIMARY KEY) |
| attach_databases | TEXT | Путь к внешним БД в формате "alias=path;alias=path" |
| creation_sql | TEXT | Полный SQL CREATE TABLE AS SELECT (нормализованный) |

**Нормализация SQL:**
- ✅ Удаляются комментарии (/* */ и --)
- ✅ Переносы строк заменяются на пробелы
- ✅ Множественные пробелы нормализуются в одиночные
- ✅ Результат хранится как одна строка для надёжности параметризации

**Пример:**
```sql
-- Исходный SQL с переносами и комментариями
/* ATTACH: external=D:\data\external.db */
CREATE TABLE report_sales AS
SELECT *
FROM main.sales s
LEFT JOIN external.products p ON s.product_id = p.id;

-- Сохраняется в БД как
CREATE TABLE report_sales AS SELECT * FROM main.sales s LEFT JOIN external.products p ON s.product_id = p.id;
```

### PostgreSQL vs SQLite

**PostgreSQL:**
- Использует встроенный информационный схему (information_schema)
- Подробные комментарии хранятся в pg_description
- Автоматический анализ типов через pg_type

**SQLite:**
- Требует явного сохранения метаданных в system_report_metadata
- Комментарии не сохраняются в sqlite_master (поэтому нужна таблица)
- Все метаданные управляются приложением

```
1. GetAvailableReportsAsync(schemaName)
   ↓
   Система сканирует БД и возвращает список всех отчётов
   ↓
2. ExecuteReportAsync(reportInfo)
   ↓
   Система подключает внешние БД (если нужны)
   Выполняет запрос
   Возвращает результаты в DataTable
   ↓
3. (Опционально) RefreshMaterializedViewAsync(reportInfo)
   ↓
   Пересоздаёт материализованное представление
   Обновляет кэшированные данные
```

## Использование в коде (C#)

### Простой пример

```csharp
// Инъекция зависимостей
private readonly IReportService _reportService;

// Получить список отчётов
var reports = await _reportService.GetReportsListAsync();

// Выполнить отчёт
var reportInfo = reports.First(r => r.Name == "report_sales");
var data = await _reportService.GetReportDataAsync(reportInfo);

// Использовать результаты
foreach (DataRow row in data.Rows)
{
    Console.WriteLine($"{row["name"]}: {row["amount"]}");
}
```

### С параметрами (PostgreSQL функции)

```csharp
var reportInfo = new ReportInfo
{
    Name = "get_sales_by_period",
    Schema = "public",
    Type = "Function",
    Parameters = new List<ReportParameter>
    {
        new() { Name = "start_date", Value = new DateTime(2026, 1, 1) },
        new() { Name = "end_date", Value = new DateTime(2026, 3, 31) }
    }
};

var data = await reportsRepository.ExecuteReportAsync(reportInfo);
```

### С внешними БД (SQLite)

```csharp
var reportInfo = new ReportInfo
{
    Name = "report_merged_sales",
    Type = "MaterializedView",
    AttachDatabases = "external=D:\\data\\external_sales.db"
};

var data = await reportsRepository.ExecuteReportAsync(reportInfo);
```

## Интеграция в UI (ViewModel)

```csharp
public class ReportsControlVM : ViewModelBase
{
    private readonly IReportService _reportService;
    
    public ObservableCollection<ReportViewModel> ReportsList { get; } = new();
    public DataView ReportData { get; set; }
    
    public async Task LoadReports()
    {
        try
        {
            var reports = await _reportService.GetReportsListAsync();
            
            ReportsList.Clear();
            foreach (var report in reports)
            {
                ReportsList.Add(new ReportViewModel
                {
                    Name = report.Name,
                    Type = report.Type,
                    Description = report.Description,
                    Schema = report.Schema
                });
            }
        }
        catch (Exception ex)
        {
            MessageBox.Show($"Ошибка загрузки отчётов: {ex.Message}");
        }
    }
    
    public async Task ExecuteSelectedReport(ReportViewModel report)
    {
        try
        {
            var reportInfo = new ReportInfo 
            { 
                Name = report.Name,
                Type = report.Type,
                Schema = report.Schema
            };
            
            var dataTable = await _reportService.GetReportDataAsync(reportInfo);
            ReportData = new DataView(dataTable);
        }
        catch (Exception ex)
        {
            MessageBox.Show($"Ошибка выполнения отчёта: {ex.Message}");
        }
    }
    
    public async Task RefreshSelectedReport(ReportViewModel report)
    {
        if (report.Type != "MaterializedView")
        {
            MessageBox.Show("Только материализованные представления можно обновлять");
            return;
        }
        
        try
        {
            var reportInfo = new ReportInfo 
            { 
                Name = report.Name, 
                Type = "MaterializedView" 
            };
            
            await _reportService.RefreshReportAsync(reportInfo);
            MessageBox.Show("Отчёт успешно обновлён");
        }
        catch (Exception ex)
        {
            MessageBox.Show($"Ошибка обновления: {ex.Message}");
        }
    }
}
```

## Обработка ошибок

### Типовые ошибки

| Ошибка | Причина | Решение |
|--------|---------|---------|
| Report not found | Отчёт не найден в БД | Проверить наличие представления/функции |
| Invalid schema | Схема не существует | Указать правильное имя схемы |
| Parameter missing | Не передан обязательный параметр | Заполнить все параметры функции |
| Database locked | БД заблокирована другим процессом | Подождать и повторить |
| Access denied | Нет прав доступа | Проверить права SQL пользователя |

### Пример обработки

```csharp
try
{
    var data = await reportsRepository.ExecuteReportAsync(reportInfo);
}
catch (ArgumentException ex) when (ex.Message.Contains("not found"))
{
    _logger.LogWarning($"Отчёт не найден: {reportInfo.Name}");
    // Показать user-friendly сообщение
}
catch (InvalidOperationException ex) when (ex.Message.Contains("parameter"))
{
    _logger.LogError($"Ошибка параметров: {ex.Message}");
    // Предложить заполнить параметры
}
catch (Exception ex)
{
    _logger.LogError(ex, "Неожиданная ошибка при выполнении отчёта");
    // Общая ошибка
}
```

## Производительность

### Рекомендации

| Сценарий | Рекомендация | Тип |
|----------|-------------|-----|
| Частые чтения, редко меняются | Материализованное представление | MaterializedView |
| Частые чтения, часто меняются | Обычное представление | View |
| Большой объём данных | Материализованное представление | MaterializedView |
| Параметризованные запросы | Функция | Function |
| Сложная бизнес-логика | Функция | Function |

### Оптимизация

```sql
-- Плохо: отбирает всё
SELECT * FROM report_large;

-- Хорошо: фильтрует данные
SELECT id, name, amount FROM report_large WHERE amount > 1000;

-- Плохо: медленное выполнение
CREATE TABLE report_complex AS
SELECT * FROM huge_table WHERE process complex logic;

-- Хорошо: с индексами
CREATE INDEX idx_report_complex_date ON report_complex(date_column);
```

## Документирование

В системе отчёты могут быть задокументированы через комментарии в БД:

### PostgreSQL

```sql
-- Добавить описание представлению
COMMENT ON VIEW public.report_sales IS 'Отчёт о продажах по периодам';

-- Добавить описание функции
COMMENT ON FUNCTION public.get_sales_by_region(date, date) IS 'Получить продажи по регионам за период';
```

### SQLite

SQLite не хранит комментарии в schema. Используйте таблицу метаданных:

```sql
CREATE TABLE report_metadata (
    report_name TEXT PRIMARY KEY,
    description TEXT,
    attach_databases TEXT
);
```
