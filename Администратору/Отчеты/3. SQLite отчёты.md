# SQLite отчёты

## Обзор

SQLite поддерживает два основных типа отчётов:
- **View** — обычные представления (CREATE VIEW)
- **MaterializedView** — материализованные представления (CREATE TABLE AS SELECT)

SQLite также имеет уникальную возможность подключения внешних БД через **ATTACH DATABASE** для выполнения кросс-БД запросов.

## Типы отчётов в SQLite

### 1. View (Обычное представление)

```sql
-- Создание
CREATE VIEW report_sales AS
SELECT 
    o.id,
    o.order_date,
    c.customer_name,
    p.product_name,
    od.quantity,
    od.unit_price,
    (od.quantity * od.unit_price) as total_amount
FROM orders o
JOIN customers c ON o.customer_id = c.id
JOIN order_details od ON o.id = od.order_id
JOIN products p ON od.product_id = p.id
WHERE o.order_date >= date('now', '-90 days')
ORDER BY o.order_date DESC;
```

**Особенности:**
- ✅ Не занимает место на диске
- ✅ Всегда актуальные данные
- ✅ Создание выполняется мгновенно
- ❌ Медленнее при большом объёме
- ❌ Без встроенного кэширования

### 2. MaterializedView (Материализованное представление)

В SQLite нет встроенного синтаксиса MATERIALIZED VIEW (как в PostgreSQL), поэтому используется обычная таблица:

```sql
-- Создание таблицы-отчёта
CREATE TABLE report_sales_summary AS
SELECT 
    DATE(o.order_date) as sale_date,
    c.region,
    p.category,
    COUNT(DISTINCT o.id) as transaction_count,
    COUNT(DISTINCT c.id) as unique_customers,
    SUM(od.quantity) as total_quantity,
    SUM(od.quantity * od.unit_price) as total_amount,
    AVG(od.quantity * od.unit_price) as avg_transaction
FROM orders o
JOIN customers c ON o.customer_id = c.id
JOIN order_details od ON o.id = od.order_id
JOIN products p ON od.product_id = p.id
WHERE o.order_date >= date('now', '-1 year')
GROUP BY DATE(o.order_date), c.region, p.category
ORDER BY sale_date DESC, region, category;

-- Добавить индексы
CREATE INDEX idx_report_sales_date ON report_sales_summary(sale_date DESC);
CREATE INDEX idx_report_sales_region ON report_sales_summary(region);
```

**Особенности:**
- ✅ Очень быстрое выполнение
- ✅ Можно создавать индексы
- ✅ Занимает место на диске
- ❌ Требует обновления вручную
- ✅ Поддерживается система хранения метаданных для ATTACH

**Обновление:**

```sql
-- Пересоздать таблицу с новыми данными
DROP TABLE IF EXISTS report_sales_summary;
CREATE TABLE report_sales_summary AS
SELECT ... /* исходный запрос */
```

## ATTACH DATABASE

### Основной концепт

ATTACH DATABASE позволяет подключить внешнюю БД и использовать её таблицы в SQL запросах:

```sql
-- Подключить внешнюю БД
ATTACH DATABASE '/path/to/external.db' AS external;

-- Использовать таблицы из подключённой БД
SELECT p.id, p.name, e.external_data
FROM main.products p
LEFT JOIN external.external_products e ON p.id = e.product_id;

-- Отключить БД
DETACH DATABASE external;
```

### Метаданные ATTACH и SQL

Система хранит информацию об ATTACH и сохраняет SQL в таблице `system_report_metadata`:

```sql
-- Таблица метаданных создаётся автоматически
CREATE TABLE IF NOT EXISTS system_report_metadata (
    report_name TEXT PRIMARY KEY,
    attach_databases TEXT,
    creation_sql TEXT
);

-- Колонки:
-- report_name     - имя таблицы/представления (PRIMARY KEY)
-- attach_databases - внешние БД в формате "alias1=path1.db;alias2=path2.db"
-- creation_sql    - полный SQL (CREATE TABLE AS SELECT) в нормализованном виде

-- Пример:
INSERT OR REPLACE INTO system_report_metadata VALUES (
    'report_merged_sales',
    'external=D:\data\external_sales.db;archive=D:\data\archive_sales.db',
    'CREATE TABLE report_merged_sales AS SELECT s.id, s.date, e.ext_data FROM main.sales s LEFT JOIN external.ext_sales e ON s.id = e.sale_id;'
);
```

**Нормализация SQL:**
- Удаляются комментарии (/* */ и --)
- Переносы строк заменяются на пробелы
- Множественные пробелы схлопываются в одиночные
- Результат хранится как одна строка для надёжности параметризации

## Использование в коде (C#)

### Пример 1: Простой View

```csharp
var reportInfo = new ReportInfo
{
    Name = "report_sales",
    Schema = "main",
    Type = "View"
};

var data = await reportsRepository.ExecuteReportAsync(reportInfo);

// Результат: DataTable с результатами запроса
foreach (DataRow row in data.Rows)
{
    Console.WriteLine($"{row["customer_name"]}: {row["total_amount"]}");
}
```

### Пример 2: MaterializedView

```csharp
var reportInfo = new ReportInfo
{
    Name = "report_sales_summary",
    Schema = "main",
    Type = "MaterializedView"
};

// Выполнить отчёт (читает кэшированные данные)
var data = await reportsRepository.ExecuteReportAsync(reportInfo);

// Обновить отчёт (пересоздаст таблицу)
await reportsRepository.RefreshMaterializedViewAsync(reportInfo);
```

### Пример 3: С внешней БД

```csharp
var reportInfo = new ReportInfo
{
    Name = "report_merged_sales",
    Schema = "main",
    Type = "MaterializedView",
    AttachDatabases = "external=D:\\data\\external_sales.db"
};

// Система автоматически:
// 1. Подключит внешнюю БД
// 2. Выполнит SELECT из таблицы отчёта
// 3. Вернёт результаты
var data = await reportsRepository.ExecuteReportAsync(reportInfo);
```

### Пример 4: Несколько внешних БД

```csharp
var reportInfo = new ReportInfo
{
    Name = "report_consolidated",
    Schema = "main",
    Type = "MaterializedView",
    AttachDatabases = "db1=D:\\data\\sales.db;db2=D:\\data\\products.db;db3=D:\\data\\customers.db"
};

var data = await reportsRepository.ExecuteReportAsync(reportInfo);
```

## Примеры SQL

### Создание отчёта с одной внешней БД

```sql
-- Создание
CREATE TABLE report_sales_with_products AS
SELECT 
    s.id,
    s.sale_date,
    s.customer_id,
    s.amount,
    p.product_name,
    p.category,
    p.supplier
FROM main.sales s
LEFT JOIN external.products p ON s.product_id = p.id
WHERE s.sale_date >= date('now', '-30 days');

-- Сохранение метаданных с информацией об ATTACH и SQL
INSERT OR REPLACE INTO system_report_metadata VALUES (
    'report_sales_with_products',
    'external=D:\data\products.db',
    'CREATE TABLE report_sales_with_products AS SELECT s.id, s.sale_date, s.customer_id, s.amount, p.product_name, p.category, p.supplier FROM main.sales s LEFT JOIN external.products p ON s.product_id = p.id WHERE s.sale_date >= date(''now'', ''-30 days'');'
);
```

### Создание отчёта с несколькими внешними БД

```sql
-- Создание
CREATE TABLE report_consolidated_sales AS
SELECT 
    s.id,
    s.sale_date,
    p.product_name,
    p.category,
    c.customer_name,
    c.customer_type,
    (s.amount * tax.tax_rate / 100) as tax_amount,
    s.amount as sale_amount
FROM main.sales s
LEFT JOIN db1.products p ON s.product_id = p.id
LEFT JOIN db2.customers c ON s.customer_id = c.id
LEFT JOIN db3.tax_rates tax ON p.category = tax.category
WHERE s.sale_date >= date('now', '-90 days');

-- Сохранение метаданных
INSERT OR REPLACE INTO system_report_metadata VALUES (
    'report_consolidated_sales',
    'db1=D:\data\products.db;db2=D:\data\customers.db;db3=D:\data\tax_rates.db',
    'CREATE TABLE report_consolidated_sales AS SELECT s.id, s.sale_date, p.product_name, p.category, c.customer_name, c.customer_type, (s.amount * tax.tax_rate / 100) as tax_amount, s.amount as sale_amount FROM main.sales s LEFT JOIN db1.products p ON s.product_id = p.id LEFT JOIN db2.customers c ON s.customer_id = c.id LEFT JOIN db3.tax_rates tax ON p.category = tax.category WHERE s.sale_date >= date(''now'', ''-90 days'');'
);
```

### Аналитический отчёт

```sql
CREATE TABLE report_sales_analytics AS
SELECT 
    strftime('%Y-%m', s.sale_date) as month,
    p.category,
    COUNT(DISTINCT s.id) as transactions,
    COUNT(DISTINCT s.customer_id) as customers,
    SUM(s.quantity) as total_qty,
    SUM(s.amount) as total_sales,
    ROUND(AVG(s.amount), 2) as avg_transaction,
    ROUND(100.0 * SUM(s.amount) / SUM(SUM(s.amount)) OVER (PARTITION BY strftime('%Y-%m', s.sale_date)), 2) as pct_of_month
FROM main.sales s
LEFT JOIN db1.products p ON s.product_id = p.id
WHERE s.sale_date >= date('now', '-12 months')
GROUP BY month, p.category
ORDER BY month DESC, total_sales DESC;

INSERT OR REPLACE INTO system_report_metadata VALUES (
    'report_sales_analytics',
    'db1=D:\data\products.db',
    'CREATE TABLE report_sales_analytics AS SELECT strftime(''%Y-%m'', s.sale_date) as month, p.category, COUNT(DISTINCT s.id) as transactions, COUNT(DISTINCT s.customer_id) as customers, SUM(s.quantity) as total_qty, SUM(s.amount) as total_sales, ROUND(AVG(s.amount), 2) as avg_transaction, ROUND(100.0 * SUM(s.amount) / SUM(SUM(s.amount)) OVER (PARTITION BY strftime(''%Y-%m'', s.sale_date)), 2) as pct_of_month FROM main.sales s LEFT JOIN db1.products p ON s.product_id = p.id WHERE s.sale_date >= date(''now'', ''-12 months'') GROUP BY month, p.category ORDER BY month DESC, total_sales DESC;'
);
```

### С рекурсивным запросом (последовательность дат)

```sql
CREATE TABLE report_daily_sales AS
WITH RECURSIVE dates(date) AS (
    SELECT date('now', '-30 days')
    UNION ALL
    SELECT date(date, '+1 day')
    FROM dates
    WHERE date < date('now')
)
SELECT 
    d.date,
    COALESCE(COUNT(DISTINCT s.id), 0) as transactions,
    COALESCE(SUM(s.amount), 0) as daily_sales,
    p.category
FROM dates d
LEFT JOIN main.sales s ON DATE(s.sale_date) = d.date
LEFT JOIN db1.products p ON s.product_id = p.id
GROUP BY d.date, p.category
ORDER BY d.date DESC, p.category;

INSERT OR REPLACE INTO system_report_metadata VALUES (
    'report_daily_sales',
    'db1=D:\data\products.db',
    'CREATE TABLE report_daily_sales AS WITH RECURSIVE dates(date) AS (SELECT date(''now'', ''-30 days'') UNION ALL SELECT date(date, ''+1 day'') FROM dates WHERE date < date(''now'')) SELECT d.date, COALESCE(COUNT(DISTINCT s.id), 0) as transactions, COALESCE(SUM(s.amount), 0) as daily_sales, p.category FROM dates d LEFT JOIN main.sales s ON DATE(s.sale_date) = d.date LEFT JOIN db1.products p ON s.product_id = p.id GROUP BY d.date, p.category ORDER BY d.date DESC, p.category;'
);
```

### Оптимизирование запроса

```sql
-- Плохо: выбирает всё
SELECT * FROM sales;

-- Хорошо: выбирает только нужные колонки
SELECT id, customer_id, amount FROM sales WHERE sale_date >= date('now', '-30 days');

-- Плохо: без фильтра перед GROUP BY
CREATE TABLE report AS SELECT category, COUNT(*) FROM products GROUP BY category;

-- Хорошо: с условиями и лимитом
CREATE TABLE report AS 
SELECT category, COUNT(*) as cnt
FROM products 
WHERE active = 1
GROUP BY category
HAVING COUNT(*) > 10
ORDER BY cnt DESC;
```

## Нормализация SQL

### Зачем нужна нормализация

При сохранении SQL скрипта создания в таблицу `system_report_metadata` происходит автоматическая нормализация:

**Проблемы многострочного SQL:**
- ❌ Параметры SQLite неправильно обрабатывают переносы строк (\r\n)
- ❌ Комментарии ATTACH занимают место и путают логику
- ❌ Множественные пробелы увеличивают размер хранилища

**Решение - NormalizeSqlForStorage():**
- ✅ Удаляет комментарии (/* */ и --)
- ✅ Заменяет переносы строк на одиночные пробелы
- ✅ Нормализует множественные пробелы в одиночные
- ✅ Сохраняет полный SQL в однострочном формате

### Пример нормализации

**Исходный SQL:**
```sql
/* ATTACH: external=D:\data\external.db */
CREATE TABLE report_sales AS
SELECT *
FROM main.sales s
LEFT JOIN external.products p
    ON s.product_id = p.id
WHERE s.sale_date >= date('now', '-30 days')
ORDER BY s.sale_date DESC;
```

**После нормализации:**
```sql
CREATE TABLE report_sales AS SELECT * FROM main.sales s LEFT JOIN external.products p ON s.product_id = p.id WHERE s.sale_date >= date('now', '-30 days') ORDER BY s.sale_date DESC;
```

## Специфика SQLite

### Ограничения

| Ограничение | Значение | Обход |
|-----------|----------|-------|
| Максимальный размер БД | 140 ТБ | Разделить на несколько файлов с ATTACH |
| Одновременные писатели | 1 | Используйте WAL mode |
| Типы данных | 5 основных | Хранить в TEXT и преобразовывать |

### WAL Mode

```sql
-- Включить WAL mode для лучшей производительности
PRAGMA journal_mode = WAL;

-- Проверить режим
PRAGMA journal_mode;

-- Назад к стандартному режиму
PRAGMA journal_mode = DELETE;
```

### Форсирование индексов

```sql
-- Если SQLite не использует нужный индекс
SELECT * FROM report_sales 
INDEXED BY idx_report_sales_date 
WHERE sale_date > date('now', '-30 days');

-- Запретить использование индекса
SELECT * FROM report_sales 
NOT INDEXED 
WHERE sale_date > date('now', '-30 days');
```

## Обработка ошибок

### Ошибка: database is locked

```
Error: database is locked
```

**Решение:** 
- Закройте другие подключения к БД
- Увеличьте timeout подключения

```csharp
var connection = new SqliteConnection(connectionString);
connection.Open();
using var command = new SqliteCommand("PRAGMA busy_timeout = 5000;", connection);
await command.ExecuteNonQueryAsync();
```

### Ошибка: no such table: system_report_metadata

```
SQLite Error 1: 'no such table: system_report_metadata'
```

**Решение:** Система автоматически создаёт эту таблицу при первом использовании. Если ошибка повторяется:

```sql
CREATE TABLE IF NOT EXISTS system_report_metadata (
    report_name TEXT PRIMARY KEY,
    attach_databases TEXT,
    creation_sql TEXT
);
```

### Ошибка: no such table: external.table_name

```
Error: no such table: external.table_name
```

**Решение:**
- Проверить путь к внешней БД
- Убедиться, что таблица существует во внешней БД

```sql
-- Проверить подключённые БД
PRAGMA database_list;

-- Проверить таблицы во внешней БД
SELECT name FROM pragma_table_info('external');
```

CREATE TABLE report_documentation (

INSERT INTO report_documentation VALUES (

## Рекомендации

### Когда использовать View

- Данные часто обновляются
- Объём данных небольшой (< 100K записей)
- Нужны максимально актуальные данные
- Читаются редко

### Когда использовать MaterializedView

- Данные редко обновляются
- Большой объём данных (> 100K записей)
- Нужна максимальная производительность
- Читаются часто (> 100 раз в день)

### Когда использовать ATTACH

- Нужны данные из нескольких файлов БД
- Разделяете данные по периодам
- Используете архивные данные
- Интегрируете внешние источники
