# FAQ по отчётам

## Общие вопросы

### Q: Какую БД мне использовать для отчётов?

**A:** Зависит от ваших требований:

| Требование | PostgreSQL | SQLite |
|-----------|-----------|--------|
| Многопользовательский доступ | ✅ | ⚠️ WAL Mode |
| Высокая производительность | ✅ | ✅ |
| Простота развёртывания | ❌ Нужен сервер | ✅ Файл |
| Встроенные функции | ✅ Много | ⚠️ Ограничено |
| Кросс-БД запросы | ❌ FDW сложный | ✅ ATTACH |
| Материализованные представления | ✅ Native | ⚠️ CREATE TABLE AS |

**Рекомендация:**
- PostgreSQL — для корпоративных решений
- SQLite — для локальных/встроенных приложений

### Q: Как выбрать между View и MaterializedView?

**A:** 
```
View когда:
- Данные обновляются часто (несколько раз в день)
- Объём < 100K записей
- Нужны актуальные данные
- Читаются редко

MaterializedView когда:
- Данные изменяются редко (раз в день/неделю)
- Объём > 100K записей
- Нужна максимальная скорость
- Читаются часто
```

### Q: Могу ли я объединить данные из разных БД?

**A:** Да, но по-разному:

**PostgreSQL:**
```sql
-- Использовать Foreign Data Wrapper (FDW)
CREATE EXTENSION postgres_fdw;
CREATE SERVER remote_db FOREIGN DATA WRAPPER postgres_fdw
    OPTIONS (host 'remote.server', dbname 'remote_db');
```

**SQLite:**
```sql
-- Использовать ATTACH DATABASE
ATTACH DATABASE '/path/to/other.db' AS other_db;
CREATE TABLE report AS
SELECT * FROM main.table1
JOIN other_db.table2 ON ...;
```

### Q: Система не находит мой отчёт

**A:** Проверьте:

1. **Имя таблицы:**
   ```sql
   -- PostgreSQL
   SELECT * FROM information_schema.tables 
   WHERE table_name = 'report_myreport';
   
   -- SQLite
   SELECT * FROM sqlite_master 
   WHERE type='table' AND name='report_myreport';
   ```

2. **Права доступа:**
   ```sql
   -- PostgreSQL
   SELECT * FROM information_schema.role_table_grants 
   WHERE table_name='report_myreport';
   ```

3. **Имя не начинается с `report_`:**
   ```sql
   -- Система ищет только report_* таблицы/представления
   -- Переименуйте если нужно
   ALTER TABLE myreport RENAME TO report_myreport;
   ```

## Производительность

### Q: Отчёт выполняется слишком медленно

**A:** Пошаговая диагностика:

```csharp
// 1. Логирование времени
var stopwatch = Stopwatch.StartNew();
var data = await reportsRepository.ExecuteReportAsync(reportInfo);
stopwatch.Stop();
_logger.LogInformation($"Время выполнения: {stopwatch.ElapsedMilliseconds}ms");
```

```sql
-- 2. Анализ плана выполнения
-- PostgreSQL
EXPLAIN ANALYZE SELECT * FROM report_myreport WHERE ...;

-- SQLite
EXPLAIN QUERY PLAN SELECT * FROM report_myreport WHERE ...;
```

```sql
-- 3. Добавить индексы на часто фильтруемые колонки
CREATE INDEX idx_report_date ON report_myreport(date_column DESC);
CREATE INDEX idx_report_category ON report_myreport(category);
```

```sql
-- 4. Пересоздать статистику
-- PostgreSQL
ANALYZE report_myreport;

-- SQLite
ANALYZE;
```

### Q: MaterializedView занимает слишком много места

**A:**

```sql
-- Посмотреть размер
-- PostgreSQL
SELECT pg_size_pretty(pg_total_relation_size('report_myreport'));

-- SQLite
SELECT page_count * page_size as size FROM pragma_page_count(), pragma_page_size();
```

**Решение:**
- Фильтрация данных: WHERE условие
- Выбор только нужных колонок
- Архивирование старых данных
- Секционирование (PostgreSQL)

## Параметризованные отчёты (PostgreSQL)

### Q: Как создать отчёт с параметрами?

**A:** Создайте функцию:

```sql
CREATE OR REPLACE FUNCTION public.get_sales(
    p_start_date DATE,
    p_end_date DATE,
    p_region VARCHAR DEFAULT NULL
)
RETURNS TABLE (
    id INTEGER,
    customer_name VARCHAR,
    total NUMERIC
) AS $$
BEGIN
    RETURN QUERY
    SELECT o.id, c.name, SUM(od.amount)
    FROM orders o
    JOIN customers c ON o.customer_id = c.id
    JOIN order_details od ON o.id = od.order_id
    WHERE o.order_date BETWEEN p_start_date AND p_end_date
      AND (p_region IS NULL OR c.region = p_region)
    GROUP BY o.id, c.name;
END;
$$ LANGUAGE plpgsql;
```

Используйте в коде:
```csharp
var reportInfo = new ReportInfo
{
    Name = "get_sales",
    Schema = "public",
    Type = "Function",
    Parameters = new List<ReportParameter>
    {
        new() { Name = "p_start_date", Value = DateTime.Now.AddDays(-30) },
        new() { Name = "p_end_date", Value = DateTime.Now },
        new() { Name = "p_region", Value = "North" }
    }
};

var data = await reportsRepository.ExecuteReportAsync(reportInfo);
```

## SQLite и ATTACH DATABASE

### Q: Как правильно использовать ATTACH DATABASE?

**A:**

```csharp
// 1. Создать отчёт с ATTACH
var reportInfo = new ReportInfo
{
    Name = "report_merged",
    Type = "MaterializedView",
    Schema = "main",
    AttachDatabases = "external=D:\\data\\external.db"  // ← Полный путь!
};

// 2. SQL в отчёте использует alias
// CREATE TABLE report_merged AS
// SELECT * FROM main.sales s
// LEFT JOIN external.products p ON s.product_id = p.id;

// 3. Метаданные автоматически сохраняются
```

### Q: Несколько внешних БД — как это работает?

**A:**

```sql
-- В SQL используйте разные alias
CREATE TABLE report_consolidated AS
SELECT 
    s.id,
    p.name,
    c.customer_name
FROM main.sales s
LEFT JOIN db1.products p ON s.product_id = p.id
LEFT JOIN db2.customers c ON s.customer_id = c.id;
```

```csharp
// В коде разделяйте точкой с запятой
reportInfo.AttachDatabases = "db1=D:\\data\\products.db;db2=D:\\data\\customers.db";
```

### Q: Путь к БД содержит пробелы

**A:** Используйте экранирование:

```csharp
// Правильно
reportInfo.AttachDatabases = "db=D:\\My Data\\databases\\external.db";

// Также правильно (прямые слеши)
reportInfo.AttachDatabases = "db=D:/My Data/databases/external.db";

// Неправильно (одинарный слеш - экранирование нарушено)
reportInfo.AttachDatabases = "db=D:\My Data\external.db"; // ❌
```

## Обновление отчётов

### Q: Как автоматически обновлять MaterializedView?

**A:** Несколько вариантов:

**1. Фоновый сервис в приложении:**
```csharp
public class ReportRefreshBackgroundService : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken stoppingToken)
    {
        while (!stoppingToken.IsCancellationRequested)
        {
            var reports = await _reportService.GetReportsListAsync();
            foreach (var report in reports.Where(r => r.Type == "MaterializedView"))
            {
                await _reportService.RefreshReportAsync(report);
            }
            await Task.Delay(TimeSpan.FromHours(1), stoppingToken);
        }
    }
}
```

**2. PostgreSQL — задача cron:**
```sql
-- Требует pg_cron extension
CREATE EXTENSION pg_cron;

SELECT cron.schedule('refresh-materialized-views', 
    '0 1 * * *',  -- В 1:00 каждый день
    'REFRESH MATERIALIZED VIEW CONCURRENTLY public.report_sales_summary'
);
```

**3. SQLite — запланированная задача (Windows):**
```batch
:: Создать задачу Windows Scheduler
schtasks /create /tn "RefreshReports" /tr "C:\path\to\RefreshReports.exe" /sc DAILY /st 01:00
```

### Q: Отчёт заблокирован, не могу обновить

**A:** PostgreSQL:

```sql
-- Посмотреть блокирующие процессы
SELECT * FROM pg_stat_activity 
WHERE datname = current_database() AND pid <> pg_backend_pid();

-- Завершить процесс
SELECT pg_terminate_backend(pid) FROM pg_stat_activity 
WHERE usename = 'your_user';
```

SQLite:

```csharp
// Увеличить timeout
var connectionString = "Data Source=path\\to\\db.db;Command Timeout=60;";
var connection = new SqliteConnection(connectionString);

// Или через pragma
using var cmd = new SqliteCommand("PRAGMA busy_timeout = 5000;", connection);
```

## Ошибки и решения

### Q: Ошибка "Report not found"

**A:** 

```
Причины:
1. Неправильное имя (регистр важен)
2. Имя не начинается с report_
3. Схема указана неправильно

Решение:
```sql
-- Найти точное имя
-- PostgreSQL
SELECT * FROM information_schema.tables 
WHERE table_schema='public' AND table_name LIKE '%report%';

-- SQLite
SELECT name FROM sqlite_master 
WHERE type='table' AND name LIKE '%report%';
```

### Q: Ошибка "Database is locked" (SQLite)

**A:**

```csharp
// 1. Увеличить timeout
var connection = new SqliteConnection(connectionString);
connection.Open();
using var cmd = new SqliteCommand("PRAGMA busy_timeout = 10000;", connection);
await cmd.ExecuteNonQueryAsync();

// 2. Закрыть другие подключения
// Убедитесь, что БД не открыта в другом приложении

// 3. Включить WAL mode
using var cmd = new SqliteCommand("PRAGMA journal_mode = WAL;", connection);
await cmd.ExecuteNonQueryAsync();
```

### Q: Ошибка "no such table: external.table_name"

**A:**

```sql
-- 1. Проверить подключённые БД
PRAGMA database_list;

-- 2. Проверить таблицы во внешней БД
SELECT name FROM sqlite_master 
WHERE type = 'table' AND db = 'external';

-- 3. Проверить путь к файлу
ATTACH DATABASE 'D:\correct\path\external.db' AS external;
```

### Q: После обновления отчёта данные потеряны

**A:** Это происходит, если SQL в метаданных неполный (нет SELECT части).

**Диагностика:**
```sql
-- Проверить, что сохранено в метаданных
SELECT creation_sql FROM system_report_metadata 
WHERE report_name = 'report_myreport';

-- Если видите только "CREATE TABLE report_myreport" без SELECT - это проблема!
```

**Решение - восстановить вручную:**

1. **Найти исходный SQL:**
   ```sql
   -- Если есть старая копия или документация
   SELECT * FROM report_myreport LIMIT 1;  -- Посмотреть структуру
   ```

2. **Обновить метаданные:**
   ```sql
   UPDATE system_report_metadata 
   SET creation_sql = 'CREATE TABLE report_myreport AS SELECT * FROM main.sales WHERE date > ''2025-01-01'';'
   WHERE report_name = 'report_myreport';
   
   -- Проверить
   SELECT creation_sql FROM system_report_metadata WHERE report_name = 'report_myreport';
   ```

3. **Пересоздать таблицу:**
   ```sql
   -- Если уже потеряны данные, пересоздайте с полным SQL
   DROP TABLE IF EXISTS report_myreport;
   
   -- Новый SQL с SELECT
   CREATE TABLE report_myreport AS 
   SELECT * FROM main.sales 
   WHERE date > '2025-01-01';
   ```

**Профилактика:**
- ✅ Новый код сохраняет нормализованный creation_sql автоматически
- ✅ Система логирует первые 150 символов сохранённого SQL
- ✅ Проверяйте логи после обновления отчёта

## Документирование

### Q: Как документировать отчёты?

**A:**

**PostgreSQL:**
```sql
COMMENT ON VIEW public.report_sales IS 'Отчёт о продажах';
COMMENT ON FUNCTION public.get_sales(DATE, DATE) IS 'Функция для получения продаж';
```

**SQLite:**
```sql
CREATE TABLE report_documentation (
    name TEXT PRIMARY KEY,
    description TEXT,
    usage TEXT,
    owner TEXT,
    created_date DATE,
    updated_date DATE
);
```

### Q: Как мониторить производительность отчётов?

**A:**

```csharp
// Добавить логирование
_logger?.Information($"Выполнение отчёта {report.Name}");
var sw = Stopwatch.StartNew();

try
{
    var data = await reportsRepository.ExecuteReportAsync(report);
    sw.Stop();
    
    _logger?.Information(
        $"Отчёт {report.Name} выполнен за {sw.ElapsedMilliseconds}ms, строк: {data.Rows.Count}");
}
catch (Exception ex)
{
    sw.Stop();
    _logger?.Error(ex, $"Ошибка отчёта {report.Name} ({sw.ElapsedMilliseconds}ms)");
}
```
